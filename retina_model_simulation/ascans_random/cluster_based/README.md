# Cluster-based Random Retinal A-scan Generator

This folder contains the **cluster-based anatomical model** for generating synthetic retinal A-scans.  
This model aims to reproduce the depth-dependent scattering characteristics of real retinal tissue by incorporating anatomical knowledge, cellular density, and layer-specific microstructure.

---

## Anatomically inspired modelling

Instead of assigning an entirely random refractive index to each pixel, the retina is modelled using **layer-specific scattering behaviour** derived from retinal histology.

Each anatomical layer (BG, RNFL, GCL, IPL, INL, OPL, ONL, ELM, IS, IS/OS junction, OS, RPE, BG) is generated by considering:

### • Scatterer density and cell-size characteristics  
Different layers contain different dominant cell types (nerve fibers, ganglion cells, bipolar cells, nuclei, photoreceptor segments, etc.).  
Scatterer concentration and variability in each layer follow these anatomical differences.

### • Formation of refractive-index clusters  
Within each layer, regions of locally higher scattering (e.g., dense cell groups or synaptic regions) are modelled as **clusters**.  
Cluster formation is controlled by the probability parameter **`p_cluster_full`**, which determines how often a layer exhibits a bundled or locally dense scattering region.

### • Background (environment) pixels  
Between clusters, the layer contains lower-scattering background regions representing extracellular space or lower-density tissue.

### • Optional bright spots  
Some layers (such as GCL, IPL, OPL) may contain **high-reflectivity spots**, introduced probabilistically to simulate strong anatomical reflectors or local high-density structures.

This approach produces depth profiles that more realistically mimic the **biological variability** of retinal layers.

---

## Script behaviour

The main script in this folder:

- Defines anatomical layer thickness ranges (`Depth_ranges`) for **Normal** and **Abnormal** retinas.
- Samples a random subset of layers (`num_layers`) to introduce anatomical variation.
- Converts physical thicknesses to a pixel grid using `n_pixel_size`.
- Builds per-layer refractive-index and thickness arrays (`refidx_cells`, `thick_cells`) according to the anatomical scattering rules.
- Concatenates these into:
  - `LD` → total layer thickness array  
  - `LN` → total refractive-index profile  

The OCT signal is then generated using the core physics modules:

- `Reference_Mirror` for reference arm modelling  
- `General_Multilayer_Fresnel_V11` for multilayer Fresnel reflection  
- Spectrometer sampling via `sbw_convertor`  
- A-scan reconstruction using `OCT_Analyse`

---

## Key parameters

- **`sample_type`**  
  `"Normal"` or `"Abnormal"` — selects anatomical thickness distributions.

- **`N_1`**  
  Baseline refractive index per retinal layer.

- **`DN_cluster`, `dn_all`**  
  Magnitude of refractive-index variation within and outside clusters.

- **`p_cluster_full`**  
  Probability of forming a refractive-index cluster in each layer  
  → higher values represent layers with naturally denser tissue structures.

- **`cluster_size_range_full`, `env_after_cluster_range_full`**  
  Control the size of clusters and the length of background regions.

- **`bright_spot_layers`, `bright_spot_prob_layers`**  
  Specify where and how often bright, high-scattering regions appear.

---

## Output

For each iteration in the loop (`num_smpl`):

- A random anatomical configuration is generated.
- The A-scan is computed and reconstructed.
- If total depth falls within the expected physical range, the script saves:

- `OPL_2` → cumulative optical-path-length per layer  
- `Depth` → reconstructed A-scan intensity (first 200 samples)

Outputs are stored as `.mat` files under the selected `sample_type` path.

---

## Applications

The generated A-scans can be used for:

- Training deep-learning models for OCT layer detection or enhancement  
- Benchmarking OCT signal-processing pipelines  
- Constructing physics-informed retinal datasets  
- Testing inversion or super-resolution algorithms  

---

This cluster-based model provides a realistic, anatomically grounded alternative to simple pixel-wise random models, enabling richer and more biologically meaningful synthetic data.
